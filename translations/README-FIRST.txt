1. Folder name / language code
--------------------------------

Create one folder for your language inside this translations folder. Use the
exact BCP 47 language tag used by Teamfight Manager 2 and its champion.i18n
files. Keep the spelling, hyphens, and capitalization shown below.

Language tags currently found in the game and champion mods:

  German:                  de
  English:                 en
  Spanish (Spain):         es-ES
  French:                  fr
  Hawaiian:                haw
  Italian:                 it
  Japanese:                ja
  Korean:                  ko
  Dutch:                   nl
  Polish:                  pl
  Brazilian Portuguese:    pt-BR
  Russian:                 ru
  Thai:                    th
  Turkish:                 tr
  Vietnamese:              vi
  Simplified Chinese:      zh-hans
  Traditional Chinese:     zh-hant

The app can use another valid language tag too, but automatic mod-champion name
generation only works when the mod's champion.i18n contains the same tag.

English UI text is built into the app, so translators do not need to create an
en/base.json. The app may create en/mod.json when English is selected.


2. Required folder layout
--------------------------

Your language folder should look like this:

  translations/
    fr/
      base.json
      mod.json       (generated by the app; may not exist yet)

base.json contains the app interface and base-game champion translations.
mod.json contains names for enabled mod champions.

To start a new translation, use the app's "Export template" button. It creates:

  translations/translation-template/base.json

Rename the translation-template folder to your language tag, update $meta, and
translate the values in base.json.


3. Editing base.json
---------------------

Only translate the text on the right side of each key. Never rename the key on
the left.

  Correct:
    "settings.language.label": "Langue"

  Incorrect:
    "parametres.langue": "Langue"

The $meta block controls how the language appears in the selector:

  "$meta": { "name": "Français", "direction": "ltr" }

Use the language's native name for $meta.name. Keep direction as "ltr" for the
languages listed above.

Keep placeholders exactly as written, including their braces. Their position
may change to fit your grammar, but their names must not be translated:

  {generated}  {cached}  {failed}

For example:

  "已產生 {generated} 個頭像；{cached} 個已在快取中。"

Missing translation keys automatically fall back to English, so an incomplete
file will still work. A complete translation is preferred.


4. JSON rules
--------------

base.json must remain valid UTF-8 JSON:

  - Use double quotes around keys and text.
  - Put a comma after every entry except the final entry in an object.
  - Do not leave a trailing comma before a closing }.
  - Do not add // comments or /* comments */.
  - Escape a double quote inside translated text as \".
  - Preserve placeholders such as {generated} exactly.

An invalid base.json will be excluded from the language selector. The Settings
screen will report the filename and JSON error so it can be corrected.


5. About mod.json
-----------------

Translators normally do not need to create or edit mod.json.

When the user selects a language and clicks "Repair missing portraits", the app
reads translated champion names supplied by enabled mods and regenerates only:

  translations/<current-language>/mod.json

The file is overwritten on later repairs, so manual changes to mod.json will be
lost. If a mod champion is missing from it, the champion mod must provide that
language under the matching tag in its text/champion.i18n file.


6. Testing your translation
----------------------------

  1. Save base.json.
  2. Reopen the app's Settings screen.
  3. Select the language by its $meta.name.
  4. Check every tab, long labels, buttons, tooltips, and messages.
  5. Run "Repair missing portraits" while that language is selected to test
     automatic mod-champion names.

Pay special attention to text that becomes much longer than English and to
placeholder-based messages whose word order differs in your language.

Thank you for contributing a translation.
